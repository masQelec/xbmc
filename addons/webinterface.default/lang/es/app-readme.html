Kodi Web Interface - Chorus2

La interfaz web por defecto para Kodi.

Una gran interfaz de usuario web moderna para Kodi. Explore su música, películas o programas de televisión desde la comodidad de su navegador web propio. Puedes reproducir contenido multimedia a través de Kodi o transmitirlo en tu navegador. Funciona mejor con Chrome Pero juega bien con la mayoría de los navegadores modernos.

Sucesor de [Chorus] (https://github.com/jez500/chorus). Una reconstrucción completa utilizando Coffee Script, Backbone, Marionette y mucho, mucho más.

Poniéndolo en funcionamiento

A partir de Kodi v17, Chorus2 viene preinstalado de la caja, solo necesita habilitarlo y marcar algunas casillas.

Activando y Configurando

Kodi> Configuraciones> Servicios> Control

  • Habilitar "Permitir el control de Kodi a través de HTTP"
  • Seleccione la interfaz web
  • Seleccione "Interfaz web Kodi - Chorus2"
  • Habilitar "Permitir que los programas de este sistema controlen Kodi"
  • Habilitar "Permitir que los programas en otros sistemas controlen Kodi"

Por razones de seguridad, debe establecer un nombre de usuario y una contraseña para evitar el acceso no autorizado

Manual de instalación

Para Kodi v16 y versiones anteriores o si desea obtener la última versión lo antes posible, una instalación a través de zip es la forma más fácil de hacerlo. Toma la última versión de webinterface.default.2.X.X.zip de la página de versiones luego instálelo como este. NOTA: Chorus2 está destinado a poder usarse con la última versión de Kodi y algunas (o todas) las cosas podrían no funcionar en versiones anteriores debido a cambios en la API.

Usándolo

Dirija su navegador web a http: // localhost: 8080 - reemplace localhost con su dirección IP si lo usa de forma remota y si ha cambiado su puerto a algo que no sea 8080, asegúrese de cambiar eso también. Más información y el uso avanzado se puede encontrar en la página de Kodi Wiki.

Peticiones de características / Bugs

Agréguelos a la lista. Para errores por favor incluya la versión de Kodi, la versión del navegador web, Versión de chorus y cualquier error que se muestre en la consola. Para solicitudes de funciones, verifique el navegador API para ver si su solicitud es actualmente posible.

Transmisión

Descargo de responsabilidad: El éxito de esto depende de los formatos de archivo y de lo que admite el navegador. En general la mayoría de las cosas funcionan.

Transmisión auditiva

En la parte superior derecha hay algunas pestañas, dos de ellas se llaman Kodi y Local, así es como alternas el reproductor de la interfaz de usuario
que está controlando. En el modo Local, el logotipo y los acentos son rojo rosado, en el modo Kodi, el logotipo es el azul Kodi. Cuando tú están en un modo determinado, las acciones afectan a ese reproductor, por lo que si hace clic en Reproducir en una pista cuando está en modo Local, se reproducirá a través del navegador, igualmente, cuando en modo Kodi todos los comandos se envían a Kodi. También puede agregar medios a otros, haga clic en los botones de menú (tres puntos verticales) en la mayoría de los elementos multimedia.

Vídeo transmitido en vivo

La transmisión de video a través de HTML5 funciona, realmente depende del codec utilizado. También está disponible un reproductor VLC incorporado con mejor soporte de códec. Esto parece lo mejor que podemos obtener hasta que Kodi admita la transcodificación. Usuarios de Chrome: Chrome ha eliminado la compatibilidad con los complementos vlc/divx, por lo que la transmisión de un video requiere un Codec compatible con Chrome. Para obtener los mejores resultados, use Chrome con video mp4 que tenga audio de 2 canales (el audio 5.1 parece no funcionar).

Ajustes de Kodi a través de la interfaz web.

Puede cambiar la mayoría de las configuraciones que encontrará en Kodi a través de la página de configuración en la interfaz web. Algunas configuraciones se han omitido ya que requieren interacción con la GUI y otras son solo un campo de texto básico sin opciones.

Navegador Kodi API

Hay una función oculta en Chorus que te permite jugar con la API de Kodi y ver lo que es capaz a través del JSON-RPC. interfaz. Si está creando una aplicación o un complemento que utiliza la API, esto puede ser muy útil tanto para encontrar como para probar Todos los métodos y tipos disponibles. Si está pensando en una nueva función para Chorus, este es también un gran lugar para comprobar si es posible (y desarrollo de la vía rápida agregando un ejemplo de trabajo a un problema). Puedes encontrar el navegador de la API a través de "Chorus Lab" (abajo a la derecha 3 puntos verticales> "The Lab") o directamente a través de http://localhost:8080/#lab/api-browser.

Contribuyendo

Si desea mejorar este proyecto, le agradecería cualquier ayuda. Hay una rama de desarrollo para cada versión de Kodi. Por favor, realice solicitudes contra la rama dev para obtener la versión correcta (incluso mejor si puede hacer una PR para ambos). La rama de desarrollo de Leia (v18) es 18.x-dev, Krypton (v17) la rama de desarrollo es 17.x-dev. Ver en documentación de los desarrolladores para obtener información sobre com poner en marcha un entorno de desarrollo y luego compilar el proyecto utilizando un docker.
