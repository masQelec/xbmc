Traducciones

Para actualizar los archivos de idioma sólo necesita saber un poco de GIT. Esta página debería ayudar a con la estructura de los archivos de idioma.

¿Dónde están los archivos de idioma?

Hay dos lugares donde se almacenan los archivos de invalidación de idioma. El LANG_CODE son dos códigos de letra para ese idioma. EG: en, fr, de

Cadenas

src/lang/_strings/LANG_CODE.po

  • Se trata de cadenas utilizadas en toda la aplicación. En general, sólo actualizar ' msgstr '.
  • Si no hay ningún ' msgstr ' para la cadena, a continuación, copiar de en.po y actualizar, EG de.po. 
    msgctxt ""
msgid "Select a filter"
msgstr "Filter wählen"

Páginas

src/lang/LANG_CODE/PAGE.md

  • Estas son páginas completas que pueden ser anuladas con un idioma diferente.
  • Las páginas están en markdown.
  • Si no hay PAGE .md para el idioma, copie de la carpeta en y edite.
  • Solo crea una PAGE .md para una traducción completa

Añadiendo un nuevo idioma

Ejemplo: Si su nuevo idioma es French tendría un LANG_KEY de fr.

Cuéntales a la aplicación

También debe decirle a la aplicación que lo tenga como una opción. Así que edite este /src/js/helpers/translate.js.coffee y añadir fr: "French" a los idiomas en getLanguages

Duplica la estructura de carpeta/archivo de en

Copie los archivos que desea anular con el nuevo idioma:

  • Strings: copiar /src/_strings/en.po a /src/_strings/fr.po
  • Pages: copiar /src/en/readme.md a /src/fr/readme.md

Probando

Para probar necesita hacer una compilación, sin embargo, si sigue la estructura existente que no lo deberías necesitar.

Si deseas probar tu idioma en la aplicación con una compilación, puedes:

  1. Asegurarse de que nodejs, npm se instalan
  2. cd /chorus/folder
  3. npm install (sólo la primera vez)
  4. grunt lang (Esto reconstruirá sólo los idiomas en el directorio dist/lang)
  5. Actualizar Chorus

Retroceder

Las traducciones deberían cambiar al inglés a menos que msgid se establece en un archivo LANG_CODE.po. O si existe una página LANG_CODE/PAGE.md.

Enviar una actualización

Enviar una solicitud de extracción a través de [GitHub] (https://github.com/jez500/chorus2) en una nueva rama es la mejor manera. Consideraría las actualizaciones a través de otros métodos.
